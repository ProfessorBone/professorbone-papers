This file explains amendment and versioning practices by reference to the governing provisions in the Core and CRC-SSR. Those artifacts provide relevant governance context and authority for their own declared subjects; they do not delegate authority to this guide.

## Appendix B Route

Core Appendix B defines how shared apparatus is amended, how back-references work, how Core-binding changes are routed, and how compatible extensions are handled.

## Back-Reference Obligation

A lawful amendment is complete only when the amended paper carries the required back-reference. A disclosure on only the amending side is incomplete.

## CRC-SSR Update Obligation

Once CRC-SSR is active, a lawful amendment to a registered object is complete only when the same governed change set updates the defining paper, carries the required back-reference, and updates the corresponding CRC-SSR entry.

## Version Claims

A version-header or changelog change is a substantive claim. A filename or markdown-path update is navigation hygiene. Historical changelog entries are the record and are not edited.

## Option Y

Option Y remains in force. Narrow, non-breaking Core changes do not mass-update companion Core-alignment citations unless the companion text earns the changed alignment claim. The reading-order Core citation remains at v5.5 unless separately ruled.
