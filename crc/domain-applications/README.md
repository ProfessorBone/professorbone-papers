# Domain Applications

These documents show the Constitutional Runtime Computation architecture
instantiated in specific real systems under genuine engineering and operational
constraints. They are not implementation guides or tutorials. They are
architectural records — evidence that the CRC framework is not only formally
coherent but operationally deployable.

## A Note on Maturity

Both documents published here are working artifacts, not finished papers.
The Apex Build Specification in particular carries explicit stub markers
throughout its schema sections. Field definitions in those sections were
intentionally deferred pending PEAS documents that were in progress at the
time of publication.

This is disclosed rather than hidden because the architectural decisions,
governance spine, and ORSR instantiation are complete and represent the
substantive contribution. The stubs are implementation detail placeholders,
not gaps in the architecture. Readers engaging with these documents for
their architectural content will find everything they need. Readers seeking
a finished implementation reference should wait for a later version.
